# 数据库修复说明

本文件用于说明本项目历史数据库问题的处理口径。

## 重要说明

本项目**当前推荐运行路径为 MySQL**。

因此：

- 如果你遇到的是历史 SQLite 锁定、损坏、WAL 残留问题，这属于旧路径问题
- 如果你当前已经切到 MySQL，请优先参考 `MYSQL_MIGRATION.md`
- 不建议继续把 SQLite 当作长期主运行数据库

## 历史常见问题

旧版运行中，SQLite 可能出现：

- `database is locked`
- `database disk image is malformed`

这些问题通常与以下因素有关：

- 多并发写入
- 事务未及时提交
- 文件损坏
- 应用异常终止

## 当前推荐修复思路

### 场景 A：你仍在使用 SQLite

建议：

1. 先完整备份 SQLite 数据文件
2. 停止插件写入
3. 评估是否直接迁移到 MySQL
4. 如只是短期应急，再考虑 SQLite 修复

### 场景 B：你已经切到 MySQL

建议：

- 不要继续围绕 SQLite 做长期修修补补
- 以 MySQL 为主库继续运行
- 将 SQLite 保留为历史备份或迁移来源

## 如果必须修 SQLite

仅建议作为短期抢救措施：

1. 停止应用
2. 备份数据库文件
3. 做完整性检查
4. 处理 WAL/SHM 残留
5. 尽快迁移到 MySQL

## 当前项目已做的方向调整

- 插件主路径已逐步收敛到 MySQL
- 关键修复优先落在 MySQL 仓储与运行逻辑
- SQLite 相关说明保留为历史兼容文档，不再作为首选运行方案

## 推荐阅读顺序

如果你现在在处理数据库问题，请按以下顺序阅读：

1. `README.md`
2. `MYSQL_MIGRATION.md`
3. 本文件（了解历史背景）
